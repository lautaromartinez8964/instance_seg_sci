### 📖 科研扫盲班：什么是 Params、FLOPs 和 MACs？

这三个词是衡量一个模型**“到底有多大、有多重”**的黄金标准。在你要投的轻量化遥感论文里，它们将是除了 mAP 之外最重要的主角。

#### 1. Params (Parameters / 参数量) —— 模型的“体重”
*   **通俗解释：** 你的模型里有多少个需要学习的变量（比如卷积核的权重 $W$ 和偏置 $b$）。
*   **决定了什么：** 决定了模型存成 `.pth` 文件后有多大（占用多少硬盘），以及加载到显存里**静态占用**多少空间。
*   **单位：** M (Million，百万)。比如 ResNet-50 大约是 25M，加上 Mask R-CNN 的头，总共大约是 **44M** 左右。
*   **遥感意义：** 无人机或卫星上的芯片存储空间极小。你的 LightMamba 目标就是把这个 44M 压缩到 20M 左右！

#### 2. FLOPs (Floating Point Operations / 浮点运算次数) —— 模型的“运动量”
*   **通俗解释：** 显卡把一张图从输入层推到输出层（前向传播一次），总共需要做多少次加减乘除的小数点计算。
*   **决定了什么：** 决定了模型的**计算复杂度**。FLOPs 越高，算得越慢，显卡发热越严重，耗电越多。
*   **单位：** G (Giga，十亿)。一张 800x800 的图跑过 ResNet-50 + Mask R-CNN，FLOPs 大概在 **200G~300G** 之间。

#### 3. MACs (Multiply-Accumulate Operations / 乘加累积操作数) —— FLOPs 的“亲兄弟”
*   **通俗解释：** 在神经网络里，最常见的操作是算 $y = w \cdot x + b$。这就是一次乘法（Multiply）加一次加法（Accumulate）。这一个打包的动作就叫 1 个 MAC。
*   **它俩的关系：** 严格来说，1 个 MAC 包含了 1 次乘法和 1 次加法，所以 **1 MAC ≈ 2 FLOPs**。
*   *注意：在很多计算机视觉论文里，大家经常把 MACs 和 FLOPs 混着用（甚至把 MACs 直接标成 FLOPs），只要你在论文里统一标准说明即可。*

### 🛠️ MMDetection 会自动提供这些吗？

#### 👉 **1. 精度指标 (mAP 等) —— 全自动化！**
这部分**完全自动**。在每个 epoch 结束时，日志里弹出的：
*   `bbox_mAP`、`bbox_mAP_50`、`bbox_mAP_s` (小目标精度)
*   `segm_mAP`、`segm_mAP_50`、`segm_mAP_s`
这就是 TGRS 论文里那张硕大的对比表（Table 1）里的数据！MMDetection 已经帮你算得明明白白。

#### 👉 **2. Params 和 FLOPs —— 需要敲一行命令提取**
训练过程中 MMDetection **不会**自动打印 Params 和 FLOPs，因为它和具体的输入图片大小有关（图片越大，FLOPs 越高）。
等你的训练跑完（甚至现在就可以开个新终端去跑），用 MMDetection 自带的神器脚本，一键就能算出来：

```bash
cd ~/projects/mmdetection
conda activate mmdec

# 计算你的 ResNet-50 Baseline 的 Params 和 FLOPs（指定输入尺寸为 800x800）
python tools/analysis_tools/get_flops.py \
    projects/iSAID/configs/mask_rcnn_r50_fpn_1x_isaid.py \
    --shape 800 800
```
运行完，屏幕上会赫然打出类似：`FLOPs: 260 G, Params: 44.17 M` 这样的终极数据。

#### 👉 **3. 还有一个隐藏的关键指标：FPS (帧率 / 推理速度)**
TGRS 非常看重你的模型在实际应用中跑得快不快。Params 小不代表一定快。测速度也有专门的脚本：
```bash
python tools/analysis_tools/benchmark.py \
    projects/iSAID/configs/mask_rcnn_r50_fpn_1x_isaid.py \
    work_dirs/mask_rcnn_r50_fpn_1x_isaid/epoch_12.pth
```
它会让模型狂跑几百张图，然后告诉你：`Average FPS: 12.5` (每秒能处理12.5张图)。
