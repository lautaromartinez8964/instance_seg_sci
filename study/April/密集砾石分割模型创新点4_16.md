## 创新点一：边界感知混合编码器（BA-HybridEncoder）

### 1.1 设计动机

**为什么全Mamba不适合密集砾石？**

VMamba的SS2D在浅层（高分辨率特征图）具有低通平滑特性。砾石与砾石之间的接缝只有1-2像素宽，SS2D会把这条缝平滑掉，导致相邻实例在特征空间粘连。

**为什么全CNN不够？**

深层需要建立"哪块砾石是独立个体"的全局理解，CNN的局部感受野在深层容量不足。

**解法：浅层CNN守边界，深层SSM建语义。**

| 阶段 | 模块 | 职责 | 物理依据 |
|---|---|---|---|
| Stage 1-2 | ConvNeXt Block | 保留高频边缘细节 | CNN强局部归纳偏置，不平滑接缝 |
| Stage 3-4 | VMamba SS2D | 建立实例间全局关系 | SSM低通特性在低分辨率下反而有利于语义建模 |

### 1.2 子模块A：实例边界解耦（IBD, Instance Boundary Decoupling）

**放置位置**：Stage 2（CNN）输出 → Stage 3（SSM）输入的过渡点

**核心问题**：CNN能保住边缘细节，但进入SSM后边缘信息会被混合进序列状态，相邻实例的特征开始混淆。IBD在进入SSM之前，把"实例间接缝信号"显式分离并强化。

**GT生成方法**（从GT mask自动生成，无需额外标注）：

```python name=ibd_gt_gen.py
from scipy.ndimage import binary_dilation
import numpy as np

def generate_inter_instance_boundary(instance_masks, dilation_r=2):
    """
    instance_masks: (N, H, W) - 每个实例一张二值mask
    
    核心逻辑：
    把每个实例膨胀dilation_r像素，
    膨胀后与其他实例重叠的区域 = 实例间接缝边界
    
    区别于普通边缘检测：
    - 普通边缘：砾石与背景的边界（对全图密集场景无判别力）
    - 实例间边界：相邻两块砾石之间的接缝（有判别力，约占5-15%面积）
    """
    H, W = instance_masks.shape[1:]
    instance_map = np.zeros((H, W), dtype=np.int32)
    for idx, mask in enumerate(instance_masks):
        instance_map[mask > 0] = idx + 1
    
    boundary = np.zeros((H, W), dtype=np.float32)
    for idx in range(1, len(instance_masks) + 1):
        current = (instance_map == idx).astype(np.uint8)
        dilated = binary_dilation(current, iterations=dilation_r)
        other = (instance_map > 0) & (instance_map != idx)
        boundary += (dilated & other).astype(np.float32)
    
    return np.clip(boundary, 0, 1)  # (H, W), 接缝处为1
```

**IBD模块网络结构**：

```python name=ibd_module.py
class InstanceBoundaryDecoupling(nn.Module):
    """
    输入: F2 (B, C, H, W) - Stage2的CNN输出
    输出: 
        F2_enhanced (B, C, H, W) - 边界增强的特征，送Stage3
        boundary_map (B, 1, H, W) - 预测的实例间接缝图
                                     供BCRA模块和DT-FPN使用
    """
    def __init__(self, in_channels, reduction=4):
        super().__init__()
        mid_ch = in_channels // reduction
        
        # 边界特征提取器（轻量3层）
        # 最后输出1���道sigmoid，表示接缝概率
        self.boundary_extractor = nn.Sequential(
            nn.Conv2d(in_channels, mid_ch, 3, padding=1),
            nn.BatchNorm2d(mid_ch),
            nn.ReLU(inplace=True),
            nn.Conv2d(mid_ch, mid_ch, 3, padding=1, groups=mid_ch),  # DW
            nn.BatchNorm2d(mid_ch),
            nn.ReLU(inplace=True),
            nn.Conv2d(mid_ch, 1, 1),
            nn.Sigmoid()
        )
        
        # 门控融合：boundary_map控制边界信号注入强度
        self.gate_proj = nn.Conv2d(in_channels, in_channels, 1)
        # alpha从0开始学习，避免初期破坏CNN特征
        self.alpha = nn.Parameter(torch.zeros(1))
    
    def forward(self, x):
        boundary_map = self.boundary_extractor(x)  # (B,1,H,W)
        gate = torch.sigmoid(self.gate_proj(x))    # (B,C,H,W)
        boundary_gate = gate * boundary_map         # 接缝处增强，内部不变
        x_enhanced = x + torch.tanh(self.alpha) * boundary_gate * x
        return x_enhanced, boundary_map
    
    def compute_loss(self, boundary_pred, boundary_gt):
        """
        正负样本极不平衡（接缝约5-15%）→ 带pos_weight的BCE + Dice
        """
        num_pos = boundary_gt.sum() + 1e-6
        num_neg = (1 - boundary_gt).sum()
        pos_weight = torch.tensor([num_neg / num_pos]).to(boundary_pred.device)
        
        loss_bce = F.binary_cross_entropy(
            boundary_pred, boundary_gt, pos_weight=pos_weight)
        
        inter = (boundary_pred * boundary_gt).sum()
        loss_dice = 1 - (2*inter + 1) / (
            boundary_pred.sum() + boundary_gt.sum() + 1)
        
        return loss_bce + 0.5 * loss_dice
```

**监督信号的关键性质**：

| 信号 | 前景/背景监督（旧方案） | 实例间边界监督（IBD） |
|---|---|---|
| 在全图密集场景中的比例 | 95%为1，5%为0 → 无判别力 | 5-15%为1，其余为0 → 有判别力 |
| 信息量 | 极低（全是前景） | 高（只标接缝） |
| 对SSM的作用 | 无法区分相邻实例 | 明确标出哪里是实例边界 |

### 1.3 子模块B：边界条件残差注意力（BCRA, Boundary-Conditioned Residual Attention）

**放置位置**：Stage 3 SSM输出之后、Stage 4 SSM输出之后各一个

**解决的问题**：SSM处理时会把相邻实例的特征通过序列状态混合，BCRA在SSM输出之后用boundary_map**抑制跨边界的特征融合**，增强同一实例内部的特征一致性。

**为什么不是SSM内部调制**：预训练权重d_state=1，SSM隐状态只有1维，内部调制（dt/z门控）在如此有限的状态容量下无效。BCRA完全在SSM外部工作，不受d_state限制。

```python name=bcra_module.py
class BoundaryCRAttention(nn.Module):
    """
    在SSM输出之后，用实例边界图压制跨边界特征混合
    
    核心逻辑：
    - 实例内部（boundary_map≈0）：增强特征内聚性
    - 实例边界（boundary_map≈1）：保持原始特征，不做跨边界融合
    
    boundary_map来自IBD模块，resize到当前特征图分辨率
    """
    def __init__(self, dim):
        super().__init__()
        # 实例内部特征增强器（DW conv）
        self.interior_enhance = nn.Sequential(
            nn.Conv2d(dim, dim, 3, padding=1, groups=dim),
            nn.BatchNorm2d(dim),
            nn.GELU()
        )
        self.alpha = nn.Parameter(torch.zeros(1))
    
    def forward(self, ssm_feat, boundary_map):
        """
        ssm_feat: (B, C, H, W) - SSM阶段输出
        boundary_map: (B, 1, H/2, W/2) 或 (H/4,W/4) - IBD输出，需resize
        """
        # resize boundary_map到当前特征分辨率
        bmap = F.interpolate(
            boundary_map, size=ssm_feat.shape[2:], mode='bilinear', 
            align_corners=False)
        
        interior_mask = 1.0 - bmap  # 实例内部≈1，边界≈0
        
        # 实例内部增强
        interior_feat = self.interior_enhance(ssm_feat)
        enhanced = ssm_feat + interior_feat * interior_mask
        
        # 边界处用原始特征，内部用增强特征
        output = ssm_feat * bmap + enhanced * interior_mask
        
        # 残差连接，alpha从0学习
        return ssm_feat + torch.tanh(self.alpha) * (output - ssm_feat)
```

## 创新点二：距离变换引导FPN（DT-FPN）

### 2.1 设计动机

**为什么标准FPN对密集砾石不够**：

标准FPN的top-down融合中，每一层对浅层和深层特征的融合权重是固定的。但对密集砾石来说：
- **实例中心区域**：需要更多深层语义特征（识别"这是独立个体"）
- **实例边界区域**：需要更多浅层高分辨率特征（精确定位"这里是接缝"）

**为什么不用拉普拉斯HF-FPN**：砾石全图都是高频，拉普拉斯对全图均匀激活，无判别力。

**距离变换图的核心性质**：
```
对每个实例，计算每个像素到该实例最近边界的距离
→ 实例中心：距离大（值高），形成"山峰"
→ 实例边界：距离为0，形成"山谷"

这个分布天然区分了密集粘连的实例：
即使两块颜色相同的砾石，它们各自有独立的距离变换"峰"
```

### 2.2 GT生成

```python name=dt_gt_gen.py
from scipy.ndimage import distance_transform_edt
import numpy as np

def generate_distance_transform_map(instance_masks):
    """
    instance_masks: (N, H, W)
    返回: dist_map (H, W) ∈ [0, 1]
    
    每个像素的值 = 该像素到最近实例边界的归一化距离
    实例中心处最大，边界处为0
    """
    H, W = instance_masks.shape[1:]
    dist_map = np.zeros((H, W), dtype=np.float32)
    
    for mask in instance_masks:
        if mask.sum() == 0:
            continue
        dist = distance_transform_edt(mask)  # 实例内部到边界的距离
        if dist.max() > 0:
            dist = dist / dist.max()  # 归一化到[0,1]
        dist_map = np.maximum(dist_map, dist)  # 取各实例最大值
    
    return dist_map
```

### 2.3 DT-FPN网络结构

```python name=dt_fpn.py
class DistanceTransformFPN(nn.Module):
    """
    距离变换引导的FPN
    
    核心机制：
    在top-down融合时，用预测的距离变换图动态调整融合权重
    - 边界区域（dt小）→ 用更多浅层特征（高分辨率，精确边界）
    - 中心区域（dt大）→ 用更多深层特征（高语义，实例识别）
    """
    def __init__(self, in_channels_list, out_channels):
        super().__init__()
        self.lateral_convs = nn.ModuleList([
            nn.Conv2d(c, out_channels, 1) for c in in_channels_list])
        self.fpn_convs = nn.ModuleList([
            nn.Conv2d(out_channels, out_channels, 3, padding=1)
            for _ in in_channels_list])
        
        # 距离变换预测头（从最深层特征预测粗粒度DT图）
        self.dt_head = nn.Sequential(
            nn.Conv2d(out_channels, out_channels // 4, 3, padding=1),
            nn.ReLU(),
            nn.Conv2d(out_channels // 4, 1, 1),
            nn.ReLU()  # 距离变换值≥0
        )
        
        # 可学习融合权重（每层一个）
        self.fusion_weights = nn.ParameterList([
            nn.Parameter(torch.ones(1)) 
            for _ in range(len(in_channels_list) - 1)])
    
    def forward(self, features, dt_gt=None):
        """
        features: [F1, F2, F3, F4] from backbone (low→high semantic)
        dt_gt: (B, 1, H, W) 距离变换GT，训练时提供
        """
        laterals = [conv(f) for conv, f in 
                    zip(self.lateral_convs, features)]
        
        # 从最深层预测DT图（粗粒度先验）
        dt_pred = self.dt_head(laterals[-1])  # (B,1,H/32,W/32)
        
        # Top-down融合（DT引导权重）
        for i in range(len(laterals)-1, 0, -1):
            up = F.interpolate(
                laterals[i], size=laterals[i-1].shape[2:], mode='bilinear')
            
            # 上采样DT图到当前层分辨率
            dt_up = F.interpolate(
                dt_pred, size=laterals[i-1].shape[2:], mode='bilinear')
            # 归一化：实例中心≈1，边界≈0
            dt_norm = torch.sigmoid(dt_up)
            
            # 中心区域权重偏向深层特征(up)
            # 边界区域权重偏向浅层特征(laterals[i-1])
            w = self.fusion_weights[i-1] * dt_norm
            laterals[i-1] = laterals[i-1] * (1-w) + up * w
        
        outs = [conv(lat) for conv, lat in 
                zip(self.fpn_convs, laterals)]
        
        dt_loss = None
        if dt_gt is not None:
            dt_gt_down = F.interpolate(
                dt_gt, size=dt_pred.shape[2:], mode='bilinear')
            dt_loss = F.mse_loss(dt_pred, dt_gt_down)
        
        return tuple(outs), dt_pred, dt_loss
```

### 2.4 与现有HF-FPN工作的差异

| | PQGNet HF-FPN | 拉普拉斯HF-FPN | **DT-FPN（本文）** |
|---|---|---|---|
| 高频信号来源 | DWT子带 | 固定拉普拉斯算子 | **学习的距离变换图** |
| 监督信号 | 无 | 无 | **MSE（DT GT）** |
| 在密集场景的判别力 | 中 | 极低（全图激活） | **高（DT峰值区分实例）** |
| 是否区分实例内外 | 否 | 否 | **是** |
| 与实例分割的关联 | 弱 | 弱 | **强（DT从instance masks生成）** |

## 创新点三：多层级SAM2蒸馏（Multi-level SAM2 Distillation）

### 3.1 为什么SAM2蒸馏对砾石场景特别重要

砾石数据集的核心挑战：
1. **标注精度有限**：砾石边界模糊，人工标注本身就不精确
2. **数据量不足**：自建数据集规模有限，容易过拟合
3. **长尾分布**：大砾石和小碎石数量差异悬殊

SAM2的优势恰好补足这三点：
- SAM2的mask质量**远超人工标注**（亚像素级边界）
- 零样本泛化能力强，可以为每张训练图生成高质量伪标签
- 对尺度变化鲁棒（从小碎石到大砾石都能分割）

### 3.2 蒸馏目标的精确定义

**关键设计原则：不做特征空间对齐**

SAM2是ViT-H/L backbone，与VMamba的特征维度完全不同，强行对齐特征空间会引入大量投影噪声。我们只对齐**输出语义空间**（空间图），这些都是与网络架构无关的几何先验。

**三个蒸馏对齐目标**：

```
对齐目标1：实例间边界图（Boundary Alignment）
  教师来源：SAM2在砾石图上生成的instance masks → 提取inter-instance boundary
           （和IBD的GT生成方法完全相同，但来自SAM2的高质量mask）
  学生：IBD模块的boundary_pred
  Loss：L_bd = BCE(boundary_pred, boundary_teacher)
  
  优势：SAM2的mask比人工标注更精细，提供更高质量的边界监督

对齐目标2：距离变换图（DT Alignment）
  教师来源：从SAM2 masks生成的distance transform → dt_teacher
  学生：DT-FPN的dt_pred
  Loss：L_dt = MSE(dt_pred, dt_teacher)
  
  优势：SAM2 mask的DT图比人工标注的DT图更平滑、更准确

对齐目标3：最终分割软标签（Mask Soft Label）
  教师来源：SAM2为每个GT框内的砾石生成的soft mask probability
  学生：Mask RCNN的最终mask head输出
  Loss：L_mask = BCE(mask_pred, SAM2_mask.detach())
  
  优势：软标签提供边界附近的不确定性信息，比硬标签GT提供更丰富的监督
```

### 3.4 训练时的蒸馏Loss接入

```python name=distillation_loss.py
def compute_distillation_losses(
        boundary_pred,    # IBD模块预测
        dt_pred,          # DT-FPN预测  
        mask_pred,        # Mask RCNN输出
        boundary_teacher, # SAM2生成的边界图
        dt_teacher,       # SAM2生成的DT图
        mask_teacher):    # SAM2生成的soft mask
    
    # 注意：SAM2生成的boundary_teacher比人工GT更精细
    # 可以选择：
    # 方案A：用SAM2boundary替换GT boundary（全部用SAM2）
    # 方案B：GT boundary + SAM2 boundary 两个loss加权（更保险）
    
    L_boundary_distill = F.binary_cross_entropy(
        boundary_pred, boundary_teacher.detach())
    
    L_dt_distill = F.mse_loss(dt_pred, dt_teacher.detach())
    
    L_mask_distill = F.binary_cross_entropy(
        mask_pred, mask_teacher.detach())
    
    return L_boundary_distill, L_dt_distill, L_mask_distill
```

### 3.5 完整Loss函数

```python name=total_loss.py
def compute_total_loss(
        # 主检测损失（来自Mask RCNN框架）
        det_loss, seg_loss,
        # 创新点一损失
        boundary_pred, boundary_gt_human,
        # 创新点二损失
        dt_pred, dt_gt,
        # 创新点三损失（SAM2蒸馏，可选）
        boundary_teacher=None,
        dt_teacher=None,
        mask_pred=None, sam2_mask=None):
    
    L_main = det_loss + seg_loss  # 框架内置
    
    # C1: IBD边界监督（人工GT）
    L_boundary = ibd_module.compute_loss(boundary_pred, boundary_gt_human)
    
    # C2: DT-FPN距离变换监督
    L_dt = F.mse_loss(dt_pred, dt_gt) if dt_gt is not None else 0.0
    
    # C3: SAM2蒸馏（需要预推理）
    L_distill = 0.0
    if boundary_teacher is not None:
        L_distill += 0.5 * F.binary_cross_entropy(
            boundary_pred, boundary_teacher.detach())
    if dt_teacher is not None:
        L_distill += 0.3 * F.mse_loss(dt_pred, dt_teacher.detach())
    if sam2_mask is not None:
        L_distill += 0.2 * F.binary_cross_entropy(
            mask_pred, sam2_mask.detach())
    
    total = (L_main 
             + 0.3 * L_boundary  # IBD权重
             + 0.2 * L_dt        # DT权重
             + 0.1 * L_distill)  # 蒸馏权重（较小，避免喧宾夺主）
    
    return total
```

## 论文创新点贡献表述（最终版）

```
Contribution 1（BA-HybridEncoder）：
We propose a Boundary-Aware Hybrid Encoder that replaces the 
all-SSM backbone with a stage-aware design: shallow CNN stages for 
high-frequency boundary preservation and deep SSM stages for global 
semantic modeling. We introduce Instance Boundary Decoupling (IBD) 
at the CNN-SSM transition, supervised by automatically generated 
inter-instance boundary maps, and Boundary-Conditioned Residual 
Attention (BCRA) after SSM stages to suppress cross-boundary feature 
mixing without relying on SSM internal state modulation.

Contribution 2（DT-FPN）：
We propose Distance Transform FPN (DT-FPN), which replaces fixed-weight 
FPN fusion with instance-geometry-aware adaptive fusion. A lightweight 
prediction head estimates the distance transform map from deep features, 
which guides the top-down pathway to use high-resolution features at 
instance boundaries and semantic features at instance interiors.

Contribution 3（SAM2 Distillation）：
We propose a multi-level SAM2 distillation strategy that aligns the 
student network with SAM2's superior segmentation quality at three 
output levels: inter-instance boundary maps, distance transform maps, 
and final mask soft labels. This avoids direct feature-space alignment 
(which requires identical architectures) and instead uses spatially 
interpretable geometric priors as transfer targets.
```

## 一句话总结三个创新点的关系

```
IBD：告诉模型"哪里是接缝"（分割线在哪）
BCRA：告诉模型"接缝两侧不要混"（特征不要跨边界融合）  
DT-FPN：告诉模型"越靠近接缝越需要精细特征"（FPN权重自适应）
SAM2蒸馏：用更好的教师信号提升上面三个模块的预测质量

四者形成一条完整的"实例边界感知"主线，
每个模块都服务于同一个核心目标：精确分离密集粘连的砾石实例。
```
