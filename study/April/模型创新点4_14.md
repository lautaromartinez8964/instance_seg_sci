## 🚀 创新点一：FA-HybridMamba (频域感知混合 Backbone)

**【学术动机】**
医学图像领域的 PolyMamba 证实了 Mamba 作为一个线性状态机（SSM），本质上是一个“低通滤波器”，会吞噬高频细节（遥感密集小目标边界）。同时，SegMAN 等研究指出 Mamba 在极深层会因长程衰减丢失全局语义。因此，我们提出混合架构：浅层用物理高频旁路保护边界，深层用 Global Attention 锁定全局语义。

**【代码实现要求】**
我的网络共有 4 个 Stage，通道 `dims=[48, 96, 192, 384]`，深度 `depths=[2, 2, 9, 2]`。

*   **设计 A：HF-VSSBlock (用于 Stage 1, 2, 3)**
    *   **Step 1: 高频提取 (Zero-parameter)**。`x_hf = x - F.avg_pool2d(x, kernel_size=3, stride=1, padding=1)`。
    *   **Step 2: 核心 Mamba**。`y_mamba = self.ssm_core(x)` （这部分调用官方的 `v05_noz` Mamba 核心即可）。
    *   **Step 3: 语义引导的高频门控**。用 `y_mamba` 的语义来判断保留哪些高频：
        `gate_c = self.hf_gate(y_mamba)` （`hf_gate` 是一个极轻量的 bottleneck 卷积：`Conv2d(C, C//4) -> SiLU -> Conv2d(C//4, C) -> Sigmoid`）。
    *   **Step 4: 输出融合**。`out = y_mamba + self.alpha * (x_hf * gate_c)` （`alpha` 是 `nn.Parameter`，初始化 0.1）。
    *   **Step 5: 提取空间先验图**。为了给下一个创新点用，计算纯空间的高频权重图：`hf_spatial_map = gate_c.mean(dim=1, keepdim=True)`，并将这个 `hf_spatial_map` 保存下来。

*   **设计 B：Global Attention Block (仅用于 Stage 4)**
    *   将 Stage 4 的两个 Block 直接替换为最经典的 **Transformer Block (MHSA + FFN)**。
    *   因为 Stage 4 分辨率仅为 25x25（$N=625$），这里使用 $O(N^2)$ 的 Multi-Head Attention 计算代价极小，完美打破 SSM 长程衰减的魔咒。

## 🚀 创新点二：HF-FPN (跨层高频注入特征金字塔)

**【学术动机】**
传统 FPN 融合高低层特征时采用“无脑相加”，这在遥感中是灾难：深层（P5/P4）庞大且模糊的背景噪声，会瞬间淹没浅层（C2/C3）的密集小目标（如小汽车）。我们引入 HF-FPN，将 Backbone 在浅层提取的物理高频图（包含清晰的小车边界）当作“信息闸门”，引导高层特征的下发。

**【代码实现要求】**
请为我编写一个继承自 MMDetection `FPN` 的自定义类 `HF_FPN`。
*   **输入**：
    1.  `inputs`：Backbone 传来的 4 个尺度的特征图 `[C2, C3, C4, C5]`。
    2.  `hf_maps`：Backbone 的 Stage 1, 2, 3 传过来的 `[hf_spatial_map2, hf_spatial_map3, hf_spatial_map4]`（单通道，维度分别为 200x200, 100x100, 50x50）。
*   **融合逻辑（自顶向下路径）**：
    在 `for i in range(len(laterals) - 1, 0, -1):` 的循环中：
    1. 高层特征上采样：`upsampled = F.interpolate(laterals[i], size=laterals[i-1].shape[2:])`
    2. 计算当前层的高频门控：`gate = 1.0 + self.fusion_beta[i-1] * hf_maps[i-1]` （`fusion_beta` 是每一层独立的可学习标量，初始化为 0.0）。
    3. 门控残差融合：`laterals[i-1] = laterals[i-1] + upsampled * gate`。
*   **物理意义**：只在有高频密集目标的地方，允许高层语义大量注入；在平滑的背景区（`hf_map ≈ 0`，`gate ≈ 1`），退化为标准 FPN，阻止噪声污染。

### 🌊 一、 为什么 DWT 是拯救 Mamba 的“绝杀武器”？

你刚才对 DWT 的理解（LL管语义，LH/HL/HH管边缘）已经非常深刻了。但我再给你补充一个**能把审稿人看跪的“降维打击”级理论**：

**1. 痛点：传统的下采样（Patch Merging / 步长卷积）是“有损的屠夫”！**
现在的 ViT 和 Mamba，在不同 Stage 之间是怎么缩减分辨率的？是用一个 $2 \times 2$ 的卷积（步长为2）强行把图像变小。这种操作在信号学上叫**“有损降采样（Lossy Downsampling）”**，它会把遥感图像里那些本来就只有两三个像素的小车、小船，**不可逆地碾碎、丢弃**！这就是密集小目标漏检的物理学根源！

**2. 破局：DWT 是“完美无损的解耦魔术师”！**
DWT 最大的数学奇迹在于它是**正交变换（Orthogonal Transform）**！
一张 $H \times W$ 的特征图，经过 DWT，变成了 4 张 $\frac{H}{2} \times \frac{W}{2}$ 的图（LL, LH, HL, HH）。**没有任何一个像素的信息被丢弃！只是被分门别类地存放了！**

**3. 完美的架构分工（The Perfect Division of Labor）：**
*   **LL (低频语义图)**：干净、平滑，包含了所有的宏观上下文（港口、大海的轮廓）。**把它送给 Mamba！** 让 Mamba 在缩小了一半的分辨率上，极其丝滑、极低算力地提取长程依赖！
*   **LH + HL + HH (高频边界图)**：包含了小车的水平缝隙、垂直阴影、对角线角点。**把它全部截留下来，绕过 Mamba（HF-Bypass），直接送给你的 HF-FPN！**

**论文里的金句（直接抄）：**

#### Step 2: DWT-Mamba (最终的王炸)
一旦逻辑跑通，直接上 Haar 小波！
为了不让你去装复杂的第三方库（比如 `PyWavelets`），导师直接给你写一段**基于 PyTorch 原生张量操作、极其快、0 参数的 Haar DWT 极简代码**：

```python
import torch
import torch.nn as nn

class HaarDWT(nn.Module):
    """
    极简、极速、零参数的 Haar 小波变换
    完美适用于 Mamba/ViT 的特征解耦
    """
    def __init__(self):
        super().__init__()

    def forward(self, x):
        # x shape: (B, C, H, W)
        # 利用切片操作，将 2x2 的块拆成 4 个相位
        x0 = x[:, :, 0::2, 0::2]  # 左上
        x1 = x[:, :, 1::2, 0::2]  # 左下
        x2 = x[:, :, 0::2, 1::2]  # 右上
        x3 = x[:, :, 1::2, 1::2]  # 右下

        # 根据 Haar 小波的数学矩阵进行正交分解
        LL = (x0 + x1 + x2 + x3) / 2.0  # 低频 (主体语义)
        HL = (x0 + x1 - x2 - x3) / 2.0  # 水平高频 (垂直边缘)
        LH = (x0 - x1 + x2 - x3) / 2.0  # 垂直高频 (水平边缘)
        HH = (x0 - x1 - x2 + x3) / 2.0  # 对角线高频 (角点/噪点)

        # 把三个高频加起来（或者拼接起来），就是完美的小目标边界图！
        High_Freq_Map = torch.abs(HL) + torch.abs(LH) + torch.abs(HH) 
        
        # LL 送给下一层的 Mamba (尺寸已经自动变成 H/2, W/2 了！)
        # High_Freq_Map return 出来，送给 FPN 的门控！
        return LL, High_Freq_Map
```
