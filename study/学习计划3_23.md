## 1. 论文主线先固定

为了防止后期“边做边改题”，先把论文叙事固定成下面这条主线：

### 1.1 一句话方法定义

RS-LightMamba 是一个面向遥感实例分割的轻量级视觉状态空间框架，核心贡献包括：

- IG-Scan：区域级连续重要性排序的动态扫描机制，解决固定扫描与遥感前景稀疏分布不匹配的问题。
- Integral Distillation：Backbone + FPN 一体化结构蒸馏，训练时借助 SAM2，多尺度迁移结构知识，推理时零额外开销。
- FGEB：轻量频域边界增强模块，作为辅助模块与 L_edge 形成“增强-纠偏”闭环。

### 1.2 本文的主表编号统一

后面所有实验、日志、Excel、论文表格，统一用这套编号，不再来回改名：

- Row A：LightMamba + Mask R-CNN，标准扫描，无蒸馏，无 FGEB
- Row B：A + IG-Scan（仅降序）
- Row B'：A + IG-Scan（正/逆交替）
- Row C：A + Backbone Distillation（L_edge + L_channel）
- Row D：A + Integral Distillation（C + L_pyramid）
- Row E：B' + D
- Row F：E + FGEB（完整模型）

额外说明：

- ResNet-50、官方 VMamba-Tiny、自定义 VMamba 这些属于对比基线，不占用主消融 A-F 的编号。
- Samba 式扫描、随机排序、像素级排序、无排序等，属于“防撞车”对照，不纳入主表，单独放在扫描策略对照表。

### 1.3 论文贡献的优先级

主贡献顺序必须始终保持一致：

1. IG-Scan 是第一贡献，也是最需要证明“不是 Samba 换皮”的模块。
2. Integral Distillation 是第二贡献，必须证明 FPN 蒸馏不是可有可无。
3. FGEB 是辅助贡献，定位是“轻量增强模块”，不能写成和前两者同等级的核心创新。

## 3. 当前起点与本周任务重置

你原来的 3.23 版本已经过于简略，而且和新 idea 有两处不完全一致：

- 主扫描模块现在应统一叫 IG-Scan，不再把“前景 GT 监督版本”当作主线。
- FGEB 应放在最后联调，而不是一上来压在第二阶段，否则容易误判主创新贡献。

因此，从今天开始，整个研发顺序调整为：

1. 先把 LightMamba baseline 跑稳。
2. 再做 IG-Scan，把“排序正确性”和“与 Samba 的差异证明”做扎实。
3. 然后做蒸馏，把训练主线打通。
4. 最后接入 FGEB，作为辅助增益模块补齐完整模型。

## 5. 风险清单与应对

### 风险 1：IG-Scan 增益不显著

应对顺序：

1. 先只在 Stage 4 做验证
2. 做 identity test 和 reverse test，先排除实现错误
3. 调 G 值，优先试 G=4，再试 G=2 和 G=8
4. 检查 importance head 是否真的学到目标相关区域

### 风险 2：蒸馏无增益或不稳定

应对顺序：

1. 先只蒸 shallow + deep，不上 FPN
2. 降低蒸馏权重，增加 warmup
3. 检查教师特征与数据增强是否错位
4. 如果 SAM2 跨域太差，考虑替换更合适教师或弱化蒸馏目标

### 风险 3：FGEB 拖慢训练或引入 NaN

应对顺序：

1. 降低 alpha 初始化
2. 限制频域增强范围
3. 切换 DoG 版
4. 必要时将 FGEB 降为补充实验，不影响主稿主线

### 风险 4：论文叙事被审稿人质疑“堆料”

应对顺序：

1. 主打 IG-Scan + Integral Distillation 两大核心
2. FGEB 只作为轻量辅助项，不与前两者争主贡献位
3. 用消融表证明每个模块解决的是不同问题，而不是无脑堆模块

### 风险 5：时间线拖后

应对顺序：

1. 优先保证 A-F 主表、扫描对照表、蒸馏表
2. 泛化实验和附加可视化可以后补
3. 最晚到 2027.3 前必须保住一篇完整投稿
