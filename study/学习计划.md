```
Day 4（3.4 周二）：精读VMamba
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

     → 主要看 classification/models/vmamba.py

  √ 搞懂SSM的核心公式（这是你整个论文的数学基础）
     → 连续形式：
        h'(t) = A·h(t) + B·x(t)
        y(t) = C·h(t) + D·x(t)

     → 离散化（用ZOH）：
        Ā = exp(ΔA)
        B̄ = (ΔA)^{-1}(exp(ΔA) - I)·ΔB
        h_t = Ā·h_{t-1} + B̄·x_t
        y_t = C·h_t

     → 你需要理解：
        Δ (step size): 控制信息遗忘速度
        A: 状态转移矩阵（通常初始化为HiPPO矩阵）
        B, C: 输入/输出投影（数据相关，由x通过线性层生成）

  √ 搞懂SS2D（2D选择性扫描）
     → VMamba处理2D图像的方式：
        把H×W的特征图沿4个方向展开成1D序列
        每个方向独立做SSM
        4个方向的输出相加（或concat）融合
     → 代码位置：vmamba.py 中的 SS2D 类
     → 重点看 forward() 中的 selective_scan_fn 调用

  √ 搞懂VMamba的多尺度输出
     → VMamba和ResNet类似，有4个Stage
     → Stage 1: C=96,  H/4 × W/4
     → Stage 2: C=192, H/8 × W/8
     → Stage 3: C=384, H/16 × W/16
     → Stage 4: C=768, H/32 × W/32
     → 这4个尺度的特征会送入FPN

  √ 安装selective_scan_cuda（VMamba的CUDA加速核心）
     → cd VMamba/kernels/selective_scan
     → pip install .
     → 如果编译失败：检查CUDA版本，或者用纯PyTorch版本（慢但能跑）
```

```
     → 核心代码结构：

     from mmdet.registry import MODELS
     from mmengine.model import BaseModule
     from vmamba import VSSM  # 从VMamba源码导入

     @MODELS.register_module()
     class VMambaBackbone(BaseModule):
         def __init__(self, 
                      depths=[2, 2, 9, 2],
                      dims=[96, 192, 384, 768],
                      pretrained=None,
                      ...):
             super().__init__()
             self.vssm = VSSM(depths=depths, dims=dims, ...)
             if pretrained:
                 self.load_pretrained(pretrained)
         
         def forward(self, x):
             # VMamba的forward返回4个stage的特征
             features = self.vssm.forward_features(x)
             # 返回tuple: (stage1_feat, stage2_feat, stage3_feat, stage4_feat)
             # 维度: [(B,96,H/4,W/4), (B,192,H/8,W/8), 
             #        (B,384,H/16,W/16), (B,768,H/32,W/32)]
             return tuple(features)
```

```
     @TRANSFORMS.register_module()
     class LoadSAM2Features:
         def __init__(self, feature_dir):
             self.feature_dir = feature_dir
```

```
     class DistillMaskRCNN(MaskRCNN):
         def __init__(self, distill_cfg, **kwargs):
             super().__init__(**kwargs)
             self.distill = CrossDomainMultiScaleDistillation(**distill_cfg)
             self.lambda_distill = 0.5
         
         def loss(self, batch_inputs, batch_data_samples):
             # 原始task loss
             losses = super().loss(batch_inputs, batch_data_samples)
             
             # 获取学生特征（从backbone的forward中间提取）
             student_feats = self.backbone.get_intermediate_features()
             
             # 获取教师特征（从data_samples中读取）
             teacher_feats = self._get_teacher_features(batch_data_samples)
             
             # 蒸馏loss
             l_edge, l_chan = self.distill(student_feats, teacher_feats)
             losses['loss_edge_distill'] = self.lambda_distill * l_edge
             losses['loss_channel_distill'] = self.lambda_distill * l_chan
             
             return losses
```

## 风险应对预案

```
风险①：VMamba集成MMDet失败（3月第2周）
  严重程度：🔴 高
  应对：
    → 先检查是否有现成的VMamba+MMDet集成代码（GitHub搜索）
    → 如果实在搞不定selective_scan_cuda → 用纯PyTorch实现的SSM
    → 最坏情况：用Swin Transformer代替VMamba做Baseline
      （Swin和VMamba架构类似，MMDet原生支持Swin）
      然后在Swin上做IG-Scan的改造（把Window Attention换成SSM+IG-Scan）
  最大延迟：1周

风险②：蒸馏无效果（3月底-4月初）
  严重程度：🟡 中
  应对：
    → 检查教师特征质量（可视化SAM2特征）
    → 如果SAM2特征对遥感不好 → 换教师为在iSAID上训练好的
      ResNet-101 + MaskRCNN（先训一个大模型当教师）
    → 尝试直接MSE蒸馏（不用边缘/通道Loss），看是不是Loss设计的问题
    → 调整λ的范围：从0.01试到10.0
  最大延迟：2周

风险③：FG-IG-Scan反而降低了性能（4月）
  严重程度：🟡 中
  应对：
    → 检查排序和逆排序是否正确（用全1张量测试）
    → 减小辅助Loss权重μ（可能梯度冲突）
    → 尝试只在Stage 4使用（减小影响范围）
    → 最坏情况：放弃IG-Scan，论文改为只有两个创新点
      （RS-AFM + 多尺度蒸馏），仍然足够发JSTARS
  最大延迟：2周

风险④：完整模型性能不如预期（5月）
  严重程度：🟡 中
  应对：
    → 增加训练epoch（24→36→48）
    → 增加数据增强（Mosaic, MixUp等）
    → 稍微增大LightMamba的容量（通道[48,96,192,384]→[64,128,256,512]）
    → 只要性能比Baseline B有明显提升（>3个点），论文就能写
  最大延迟：1周

风险⑤：论文写作进度慢（6-7月）
  严重程度：🟢 低

    → Introduction让AI帮你列提纲，你来填充
    → 如果8月写不完 → 延期到9月投稿（仍然在一年内）
  最大延迟：1个月
```
