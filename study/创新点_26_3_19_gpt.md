## 2. 研究动机（统一叙事线）

### 2.1 遥感实例分割的三重困境

| 困境 | 具体表现 | 当前方案的不足 |
|---|---|---|
| **小目标密集** | 车辆群、船群，单体面积小（<32²像素）、间距近 | Transformer O(N²) 太慢；SAM 600M 参数部署困难 |
| **背景主导** | 海面、植被、沙漠等大面积低信息量区域占 85-95% 像素 | Mamba 固定扫描让 SSM 状态被背景"淹没" |
| **频域特殊性** | 遥感高频信息（建筑边缘、车辆轮廓）和高频噪声（海浪、植被纹理）共存 | 普通卷积无法区分"有用高频"和"噪声高频" |

### 2.2 为什么选 Mamba + 蒸馏

- **Mamba**：O(N) 线性复杂度，天然适合遥感大图
- **蒸馏**：训练时借用 SAM2 的 600M 参数知识，推理时 SAM2 完全移除，零额外开销

### 2.3 核心矛盾

> **Mamba 的线性递推本质使其对 token 处理顺序敏感，但现有视觉 Mamba 的扫描顺序与遥感图像的有效信息分布严重不匹配。**

## 4. 创新点①：IG-Scan（核心架构创新）

### 4.1 SSM 递推的状态衰减问题

SSM 的本质是因果递推系统：

```
h_t = Ā·h_{t-1} + B̄·x_t
y_t = C·h_t
```

矩阵 Ā 的特征值绝对值 < 1（稳定性要求），这意味着 **早期输入对后期输出的影响会指数衰减**。在遥感中，如果序列的前 80% 都是背景 token，SSM 在处理到目标区域时，其隐状态已经被大量无关信息"占据"，对前景目标的建模能力显著下降。

### 4.2 现有扫描策略深度分析（基于源码）

#### VMamba（2024）—— 固定四方向扫描

```python
# VMamba: 行扫描、列扫描、反向行扫描、反向列扫描
xs = torch.stack([
    x.flatten(2, 3),                                    # 行扫描
    x.transpose(2, 3).contiguous().flatten(2, 3)         # 列扫描
], dim=1)
xs = torch.cat([xs, torch.flip(xs, dims=[-1])], dim=1)  # + 反向
```

**特点**：完全固定，不考虑图像内容。所有图像都用同一种扫描路径。
**问题**：在遥感中，前景目标在空间中的分布千差万别。

#### MaIR（CVPR 2025）—— 嵌套 S 形条带扫描 + Shift 机制

```python
# MaIR: 把特征图分成 scan_len 宽的条带，条带内 S 形扫描
def sscan(inp, scan_len, shift_len=0):
    B, C, H, W = inp.shape
    # 奇数条带的行翻转（形成 S 形）
    for i in range(1, (W // scan_len)+1, 2):
        inp[:,:,:, i*scan_len:(i+1)*scan_len] = \
            inp[:,:,:, i*scan_len:(i+1)*scan_len].flip(dims=[-2])
    # 子行翻转
    for hi in range(H // 2):
        for wi in range(W // scan_len):
            inp[:,:, 2*hi+1, wi*scan_len:(wi+1)*scan_len] = \
                inp[:,:, 2*hi+1, wi*scan_len:(wi+1)*scan_len].flip(dims=[-1])
```

**特点**：
- 用 S 形路径保持空间连续性（相邻 token 在 2D 上也相邻）
- shift 机制让偶数层和奇数层的条带错开半个 scan_len，扩大感受野
- **关键：scan_ids 在模型初始化时就预计算好了（`mair_ids_generate`），训练推理用同一套 ID**

**核心限制：data-independent**。给定 H × W，路径完全确定，与图像内容无关。

#### Samba（CVPR 2025）—— 显著性引导的前景/背景分割拼接

```python
# Samba: 用显著性图 gt 把 token 分成前景 (s) 和背景 (ns)
def CrossScan_saliency(x_rgb, gt):
    for b in range(B):
        non_zero_indexs = traverse_saliency_tensor(gt[b:b+1,:])
        s = torch.gather(x_rgb[b:b+1,:].view(1,C,H*W), 2, non_zero_indexs_expanded)
        ns, mask_ns = extract_non_zero_values(x_rgb[b:b+1,:] * (1-gt[b:b+1,:]))
        scan_1 = torch.cat((s, ns), -1)        # 前景拼前面
        scan_1_flip = torch.cat((s.flip(-1), ns.flip(-1)), -1)
        scan_2 = torch.cat((ns, s), -1)         # 背景拼前面
        scan_2_flip = torch.cat((ns.flip(-1), s.flip(-1)), -1)
```

**特点**：
- **确实改变了 token 顺序**（不仅是加权！）
- 用二值显著性图将 token 硬分为"前景集合"和"背景集合"
- 前景集合拼在序列前部，背景拼在后部
- 4 条扫描路径：前景→背景、背景→前景、及其反转

**核心限制**：
1. **二值硬分割**：只有"前景/背景"两个类别，没有重要性的渐变排序
2. **前景集合内部无序**：同为前景的 token 之间的相对顺序取决于 `traverse_saliency_tensor` 的遍历顺序（类似光栅扫描），而非重要性高低
3. **需要外部显著性图 gt**：依赖额外的显著性预测网络或标签
4. **逐 batch 循环**：`for b in range(B)` 的实现方式效率低

### 4.3 你的 IG-Scan：精确定位差异

#### IG-Scan 的核心设计

**Step 1：区域级重要性预测（轻量、可微、端到端）**

```python
class ImportancePredictor(nn.Module):
    def __init__(self, dim, num_groups=4):
        super().__init__()
        self.num_groups = num_groups
        self.head = nn.Sequential(
            nn.AdaptiveAvgPool2d(num_groups),   # (B,C,H,W) → (B,C,G,G)
            nn.Conv2d(dim, dim // 4, 1),
            nn.GELU(),
            nn.Conv2d(dim // 4, 1, 1),
            nn.Sigmoid()
        )
    
    def forward(self, x):
        # x: (B, C, H, W)
        scores = self.head(x)  # (B, 1, G, G)
        return scores.flatten(2)  # (B, 1, G²)
```

**Step 2：按重要性分数排序区域**

```python
# 将特征图划分为 G×G 个区域
regions = rearrange(x, 'b c (g1 h) (g2 w) -> b (g1 g2) c (h w)',
                    g1=G, g2=G)  # (B, G², C, h*w)

# 排序
sorted_idx = torch.argsort(importance_scores, dim=-1, descending=not self.reverse)
# (B, 1, G²) → 用于 gather

# 区域重排
regions_sorted = torch.gather(regions, 1, sorted_idx_expand)

# 展平为 1D 序列送入 SSM
sequence = regions_sorted.reshape(B, C, -1)  # (B, C, H*W)
```

**Step 3：SSM 处理后逆排序恢复空间**

```python
output_regions = ssm_output.reshape(B, G*G, C, -1)
reverse_idx = torch.argsort(sorted_idx)
output_original = torch.gather(output_regions, 1, reverse_idx_expand)
result = rearrange(output_original, 'b (g1 g2) c (h w) -> b c (g1 h) (g2 w)',
                   g1=G, g2=G, h=h, w=w)
```

#### 与 Samba 的精确差异（这是论文能否立住的关键）

| 维度 | Samba CrossScan_saliency | 你的 IG-Scan |
|---|---|---|
| **重排粒度** | **像素级**（每个 token 单独分类） | **区域级**（G×G=16个区域整体排序） |
| **重排依据** | 二值显著性图（前景 or 背景） | 连续重要性分数（0~1 之间的实数） |
| **排序类型** | 二值分割后拼接（partition + concat） | 全序排列（full sorting） |
| **区域内部** | 前景集合内部无语义排序 | 区域内保持原始空间连续扫描 |
| **信号来源** | 需要外部显著性 GT 或额外预测头 | 端到端学习的 importance_head |
| **实现效率** | 逐 batch 循环（`for b in range(B)`） | 纯张量操作，batch 并行 |
| **正/逆交替** | 有（scan_1 和 scan_2 是互补的） | 有（偶数 block 降序，奇数 block 升序） |

**最关键的差异——用一句话概括：**

> Samba 是"先分类再拼接"（binary partition + concatenation），丢失了类内排序信息；
> IG-Scan 是"先评分再排序"（continuous scoring + full sorting），保留了区域间的完整偏序关系，
> 且区域内保持空间连续性，避免了像素级打散导致的局部结构破坏。

**为什么这个差异很重要？**

在遥感中，目标之间也有重要性差异。比如一张港口图像：
- 大型船舶（重要性 0.95）
- 小型船舶（重要性 0.70）
- 码头设施（重要性 0.50）
- 海面背景（重要性 0.05）

Samba 只能把"大船、小船、码头"都归为"前景"混在一起，SSM 处理时大船和小船的先后无法控制。
IG-Scan 可以让 SSM 按"大船区域 → 小船区域 → 码头区域 → 海面"的顺序处理，**大船所在区域获得最新鲜的 SSM 状态**。

#### 正/逆交替的理论动机

借鉴 MaIR 的 shift 思想，但动机不同：

- **MaIR shift**：空间位置上错开半个条带，扩大感受野（geometric shift）
- **你的正/逆交替**：语义优先级上交替，获取互补视角（semantic alternation）

```
偶数 Block（降序）：
  [大船区域] → [小船区域] → [码头区域] → [海面]
  SSM 状态最新鲜时处理最重要的区域 → 精细边界特征
  
奇数 Block（升序）：
  [海面] → [码头区域] → [小船区域] → [大船区域]
  SSM 先积累全局上下文，再处理重要区域 → 丰富语义特征
```

两者在残差连接后叠加，目标区域同时获得精细局部特征和丰富语义上下文。

#### 只在 Stage 3-4 使用的严格论证

| Stage | 分辨率 | Token 数 | 区域大小 | importance_head 可靠性 | 是否使用 IG |
|---|---|---|---|---|---|
| Stage 1 | H/4 × W/4 | ~40,000 | 每区域 ~2500 token | ❌ 语义太弱 | 标准扫描 |
| Stage 2 | H/8 × W/8 | ~10,000 | 每区域 ~625 token | ❌ 语义较弱 | 标准扫描 |
| Stage 3 | H/16 × W/16 | ~2,500 | 每区域 ~156 token | ✅ 语义较强 | IG-Scan |
| Stage 4 | H/32 × W/32 | ~625 | 每区域 ~39 token | ✅ 语义很强 | IG-Scan |

深层特征的每个 token 已经具备了足够的语义判别力，importance_head 才能做出有意义的评分。

## 5. 创新点②：Integral Structure-Aware Distillation（核心训练创新）

### 5.1 为什么需要"一体化"蒸馏

**现有蒸馏方案的问题**：
- MobileSAM 式：只蒸馏 backbone 最后一层输出 → FPN 仍然"自学"，neck 层语义融合能力不足
- 普通多层 MSE：忽略浅层和深层特征的性质差异，一个 loss 函数吃不下所有

**你的观察（受 fast-iTPN 启发）**：
iTPN 的核心设计是把 backbone 特征和 FPN 特征一体化训练（PatchSplit + 跨层 concat），其本质是让多尺度特征在 backbone 内部就形成金字塔。这启发我们：**蒸馏不应止步于 backbone 输出，还应延伸到 FPN 输出**，因为 Mask R-CNN 的 RPN 和 RoI Head 直接消费的是 FPN 特征，而非 backbone 特征。

### 5.2 三层蒸馏设计

#### (a) 浅层边缘蒸馏 L_edge

**目标**：迁移 SAM2 强大的边界感知能力
**为什么用梯度域而不是原始特征域**：SAM2 在 SA-1B（自然图像）上训练，其特征的"绝对值"在遥感中可能没有直接意义，但"哪里有梯度变化（即边缘）"这个结构信息是跨域通用的。

```python
def edge_aware_loss(student_feat, teacher_feat, adapter):
    """
    student_feat: (B, C_s, H, W) —— 学生浅层特征
    teacher_feat: (B, C_t, H, W) —— 教师浅层特征（离线预提取）
    adapter: nn.Conv2d(C_s, C_t, 1) —— 通道对齐
    """
    # 1. 通道对齐
    s_aligned = adapter(student_feat)  # (B, C_t, H, W)
    
    # 2. Sobel 梯度提取（固定权重，不可学）
    grad_s = sobel_gradient(s_aligned)  # sqrt(Gx² + Gy²)
    grad_t = sobel_gradient(teacher_feat)
    
    # 3. 梯度域 MSE
    loss = F.mse_loss(grad_s, grad_t)
    return loss
```

**通俗理解**：不要求学生画得和老师一模一样，但要求学生画的"轮廓线"和老师一致。

#### (b) 深层通道蒸馏 L_channel

**目标**：迁移 SAM2 的语义通道重要性分布
**为什么用 KL 散度而不是 MSE**：深层特征的每个通道代表一种语义概念（如"建筑响应"、"水体响应"），我们不关心学生和教师的绝对数值是否一样，只关心"通道间的相对重要性排序"是否一致。

```python
def channel_distill_loss(student_feat, teacher_feat, adapter, tau=4.0):
    """
    student_feat: (B, C_s, H, W)
    teacher_feat: (B, C_t, H, W)
    """
    s_aligned = adapter(student_feat)  # (B, C_t, H, W)
    
    # 全局平均池化 → 每个通道一个标量
    s_pool = F.adaptive_avg_pool2d(s_aligned, 1).flatten(1)  # (B, C_t)
    t_pool = F.adaptive_avg_pool2d(teacher_feat, 1).flatten(1)
    
    # 温度 softmax → 通道重要性概率分布
    s_dist = F.softmax(s_pool / tau, dim=-1)
    t_dist = F.softmax(t_pool / tau, dim=-1)
    
    # KL 散度
    loss = F.kl_div(s_dist.log(), t_dist, reduction='batchmean')
    return loss
```

**τ（温度）的直觉**：τ=1 时只关注最强的 1-2 个通道；τ=4 时让所有通道的排序关系都被考虑。

#### (c) FPN 金字塔蒸馏 L_pyramid（受 iTPN 启发的新增）

**目标**：直接监督 FPN 输出层（P2-P5），弥补仅蒸馏 backbone 导致的 neck 层"断档"

```python
def pyramid_distill_loss(student_fpn_feats, teacher_fpn_feats, adapters):
    """
    student_fpn_feats: dict {P2: (B,C,H2,W2), P3: ..., P4: ..., P5: ...}
    teacher_fpn_feats: dict {P2: ..., P3: ..., P4: ..., P5: ...}
    adapters: dict {P2: Conv2d, P3: Conv2d, ...}  通道对齐
    """
    loss = 0
    weights = {2: 1.0, 3: 1.0, 4: 0.5, 5: 0.5}  # P2/P3 更重要（小目标在这里）
    
    for level in [2, 3, 4, 5]:
        s_feat = adapters[f'P{level}'](student_fpn_feats[f'P{level}'])
        t_feat = teacher_fpn_feats[f'P{level}']
        
        # 尺寸对齐（如有需要）
        if s_feat.shape != t_feat.shape:
            t_feat = F.interpolate(t_feat, size=s_feat.shape[2:], mode='bilinear')
        
        loss += weights[level] * F.smooth_l1_loss(s_feat, t_feat)
    
    return loss
```

**教师 FPN 特征如何获取？**
- 方案 A（推荐）：离线将 SAM2 特征过一个冻结的轻量 FPN，生成伪教师 FPN 特征
- 方案 B：只蒸馏 P3/P4（SAM2 特征最接近的两层），跳过 P2/P5

### 5.3 总损失

```
L_total = L_cls + L_reg + L_mask           ← Mask R-CNN 任务损失
        + λ₁ · L_edge                      ← 浅层边缘蒸馏
        + λ₂ · L_channel                   ← 深层通道蒸馏  
        + λ₃ · L_pyramid                   ← FPN 金字塔蒸馏

建议初值：λ₁=0.5, λ₂=0.5, λ₃=1.0, τ=4.0
蒸馏 loss 前 1000 iter 线性 warmup（防止训练初期不稳定）
```

### 5.4 与现有蒸馏方案的差异

| 方案 | 蒸馏层次 | Loss 类型 | 跨域鲁棒性 | FPN 监督 |
|---|---|---|---|---|
| MobileSAM | 最后一层 | MSE | ❌ 直接对齐 | ❌ |
| 通用 KD | 多层 | MSE/KL | ❌ | ❌ |
| PKD (ECCV'22) | 多层 | Pearson 相关 | 中 | ❌ |
| **你的 Integral Distillation** | **浅+深+FPN** | **边缘MSE + 通道KL + 金字塔SmoothL1** | **✅ 梯度域+分布域** | **✅** |

## 6. 创新点③：FGEB（轻量频域边界增强）—— 辅助模块

### 6.1 与 L_edge 的协同叙事

论文中不要把 FGEB 单独讲，而是嵌入到蒸馏的故事线中：

> L_edge 蒸馏告诉学生"边缘应该在哪里"（监督信号），
> FGEB 增强学生"看到边缘的能力"（输入增强）。
> 两者构成 **"增强-纠偏"闭环**：FGEB 提高浅层对高频边缘的灵敏度，L_edge 防止 FGEB 把噪声高频也放大。

### 6.2 设计

```python
class FGEB(nn.Module):
    """频域引导的边界增强模块"""
    def __init__(self, channels, init_alpha=0.1):
        super().__init__()
        # 可学习频域门控掩码
        # 参数化为一个小型 MLP，输入频域能量分布，输出增强权重
        self.freq_gate = nn.Sequential(
            nn.Linear(channels, channels // 4),
            nn.ReLU(),
            nn.Linear(channels // 4, channels),
            nn.Sigmoid()
        )
        # 可学习缩放因子（小初始化，避免训练初期不稳定）
        self.alpha = nn.Parameter(torch.ones(1) * init_alpha)
    
    def forward(self, x):
        B, C, H, W = x.shape
        
        # 1. FFT
        x_freq = torch.fft.rfft2(x, norm='ortho')  # (B, C, H, W//2+1)
        
        # 2. 计算频域能量分布
        magnitude = x_freq.abs()  # (B, C, H, W//2+1)
        energy = F.adaptive_avg_pool2d(magnitude, 1).flatten(1)  # (B, C)
        
        # 3. 预测门控权重
        gate = self.freq_gate(energy)  # (B, C)
        gate = gate.unsqueeze(-1).unsqueeze(-1)  # (B, C, 1, 1)
        
        # 4. 高频增强：抑制低频中心，增强高频边缘
        # 构造高通掩码
        center_h, center_w = H // 2, (W // 2 + 1) // 2
        high_pass = torch.ones_like(magnitude)
        r = min(H, W) // 8  # 低频半径
        high_pass[:, :, :r, :r] = 0.0  # 抑制左上角低频
        
        x_high = x_freq * high_pass
        x_high_spatial = torch.fft.irfft2(x_high, s=(H, W), norm='ortho')
        
        # 5. 门控加权融合
        out = x + self.alpha * (gate * x_high_spatial)
        return out
```

### 6.3 作用位置与开销

| 项目 | 配置 |
|---|---|
| 作用层 | Stage 1 输入后（或 Stem 输出后） |
| 参数量 | < 0.1M |
| 推理延迟 | 一次 rfft2 + irfft2（几毫秒） |
| 消融控制 | alpha 初始 0.1，可设为 0 关闭 |

### 6.4 FGEB 在论文中的定位

**不要**在贡献列表里说"我们提出了一个频域边缘增强模块"——这不够有创新性。
**应该**说：

## 7. 论文贡献段（建议表述）

> **(1)** We propose **Importance-Guided Scanning (IG-Scan)**, a data-dependent dynamic scan-ordering mechanism for visual state space models. Unlike prior fixed-path (VMamba, MaIR) or binary-partition (Samba) scanning strategies, IG-Scan performs **continuous-valued, region-level full sorting** that explicitly reindexes the chronological order of SSM state updates while preserving intra-region spatial continuity.
>
> **(2)** We design an **Integral Structure-Aware Distillation** framework that jointly distills SAM2's knowledge across backbone (edge-aware + channel-aware) and FPN (pyramid-aware) levels, achieving cross-domain knowledge transfer with domain-robust loss formulations (gradient-domain MSE + channel-distribution KL + pyramid SmoothL1).
>
> **(3)** We conduct comprehensive experiments on iSAID and NWPU VHR-10 benchmarks, demonstrating that RS-LightMamba achieves competitive accuracy with ~3% of SAM's parameters and >5× inference speedup, along with detailed ablation studies validating each component.

## 8. Related Work 中的扫描策略演进叙事

```
第一代：固定方向扫描（VMamba, 2024）
  4 条固定路径：行/列/反行/反列
  完全 data-independent

第二代 A：固定但更优的路径（MaIR, CVPR 2025）
  嵌套 S 形条带 + shift 交替
  保持空间局部性和连续性
  仍然 data-independent（路径在模型初始化时预计算）

第二代 B：内容感知的二值分割重排（Samba, CVPR 2025）
  用显著性图将 token 分为前景/背景两组
  组内无序（仍按原始扫描顺序），组间拼接重排
  data-dependent，但仅 binary partition，丢失前景内部优先级

第三代（本文）：内容感知的连续评分全排序（IG-Scan）
  区域级连续重要性评分 → 全序排列
  区域内保持空间连续扫描
  data-dependent + full ordering + local continuity
```
